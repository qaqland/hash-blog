## mkdot

观察了一下，可以考虑开个 dotfile 管理的新坑。特点是只负责安装，不负责同步备份，
因此剩下的逻辑如 `ln -s` 或 `install` 般简单。

<https://github.com/qaqland/mkdot>

同步备份是一件很复杂的事情，不应该耦合在我对 dotfile 的需求中

- 绝大部分配置文件不需要 Git 历史记录，也不应该被 Git 管理
- 部分敏感数据放置在 GitHub 等开放平台不合适或者不方便
